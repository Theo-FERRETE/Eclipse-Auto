01
Vue d'ensemble

Eclipse Auto est une application web full-stack simulant une concession automobile haut de gamme. Elle repose sur une architecture découplée : SPA React côté client, API REST Express côté serveur, et Supabase comme couche de données et d'authentification.

Stack technique

React 19 + Vite 8
Interface SPA — build ultra-rapide
React Router v7
Navigation côté client
CSS pur
Design system maison, pas de framework CSS
Express.js 5
API REST — Node.js, port 3001
Supabase
PostgreSQL + Auth JWT + Realtime
Nodemailer
Envoi d'e-mails via Gmail SMTP
PM2
Process manager — redémarre au reboot
Sharp
Conversion images en WebP (build)
VPS OVH
Ubuntu 22.04 — HTTPS Let's Encrypt

Architecture globale

1
Navigateur — charge la SPA React depuis client/dist/ servi par Express
2
React Router — gère la navigation côté client sans rechargement de page
3
Express (port 3001) — reçoit les requêtes /api/*, valide les tokens JWT, interroge Supabase
4
Supabase Cloud — PostgreSQL managé, authentification JWT, mises à jour Realtime en push vers les clients

Structure Frontend

Le frontend est une Single Page Application React construite avec Vite. Chaque page et composant est isolé dans son propre dossier avec son fichier JSX et son CSS dédié.

Pages publiques & client

client/src/pages/
Home/
Page d'accueil — hero, véhicules mis en avant (3 premiers), sections marketing
Catalogue/
Liste paginée (6/page) avec filtres, recherche texte, tri, et mises à jour en temps réel via Supabase Realtime
VehicleDetail/
Fiche détaillée — galerie d'images, spécifications, prix, statut, bouton réserver (si disponible)

05
Flux d'authentification

L'authentification repose entièrement sur Supabase Auth (JWT). Le frontend génère et stocke le token ; le backend le valide sans jamais manipuler les mots de passe.

Inscription d'un client

1
L'utilisateur remplit le formulaire Register (prénom, nom, e-mail, mot de passe)
2
Supabase Auth crée le compte dans auth.users et retourne un JWT
3
Un enregistrement est inséré dans la table profiles avec le rôle "client"
4
AuthContext met à jour l'état global — l'utilisateur est immédiatement connecté

Connexion et accès aux routes protégées

1
L'utilisateur se connecte — Supabase Auth retourne un JWT stocké en mémoire côté client
2
ProtectedRoute lit le contexte — redirige vers /login si le token est absent
3
Chaque appel API inclut le JWT dans l'en-tête : Authorization: Bearer <token>
4
Le middleware requireAuth valide le token via Supabase Admin
5
Pour les routes admin, requireAdmin vérifie en plus que profiles.role = "admin"

Base de données

Base PostgreSQL hébergée sur Supabase. Les Row Level Security (RLS) sont activées sur toutes les tables — chaque utilisateur ne peut accéder qu'à ses propres données depuis le navigateur. Le serveur contourne les RLS via la clé service_role.

Trigger PostgreSQL — Synchronisation automatique

Le trigger sync_vehicle_status_on_reservation s'exécute à chaque changement de statut d'une réservation.

1
L'admin confirme une réservation → statut passe à confirmed
2
Le trigger détecte le changement et met vehicles.status = "reserved"
3
Si la réservation est annulée → le trigger remet le véhicule à "available"
4
Supabase Realtime propage la mise à jour en temps réel dans le catalogue pour tous les visiteurs
07
Optimisation images & performance

Audit de performance réalisé en mai 2026. Les images JPEG originales pesaient entre 1 et 6 MB chacune — soit 45 MB pour 6 véhicules visibles. Elles ont été converties en WebP avec Sharp.

Résultats de la conversion WebP

FichierAvantAprèsGain
Audi-RS3.jpg6.2 MB211 KB−97%
Mercedes-AMG-GT.jpg5.9 MB88 KB−98%
Lamborghini_Aventador_SVJ.jpg5.3 MB351 KB−93%
Moyenne sur 21 fichiers~3 MB~190 KB−94%
Impact : 45 MB → 1.5 MB pour 6 véhicules visibles. Chargement 4G : 2-3s → < 500ms.

optimizeImageUrl() — gestion par source

La fonction optimizeImageUrl(url, width, quality) dans lib/utils.js adapte l'URL selon la source :

SourceTraitement
Supabase StorageRedirige vers /render/image/ avec paramètres width, quality et format=webp
UnsplashAjoute les paramètres imgix : ?w=...&q=...&fm=webp&fit=crop
Local /img/Remplace l'extension .jpg → .webp automatiquement

Google Fonts — chargement non bloquant

L'ancien @import url(...) dans le CSS était découvert après le parsing du CSS, bloquant le rendu. Remplacé par des <link rel="preconnect"> dans le <head> — découverts immédiatement au parsing HTML.

Déploiement

Architecture de production

1
Navigateur — requête HTTPS vers eclipse-auto.theo-ferrete.fr
2
Express (port 3001) géré par PM2 — sert le frontend statique depuis client/dist/ et traite les requêtes /api/*
3
Fallback SPA — toute route non trouvée retourne index.html pour que React Router gère la navigation
4
Supabase Cloud — PostgreSQL + Auth hébergés externalement

Procédure de mise en production

1
Développement local — commit et push sur la branche main
2
Sur le VPS : git fetch origin && git reset --hard origin/main
3
Rebuild du frontend : cd client && npm install && npm run build
4
Redémarrage du serveur : cd server && npm install && pm2 restart eclipse-auto-api
Fichier .env.example : versionné dans Git — liste toutes les variables sans valeurs réelles. Le .env avec les secrets est exclu via .gitignore.
