01 — Contexte
Présentation du projet

Contexte et origine

Eclipse Auto est un projet réalisé dans le cadre de la formation Développeur Web et Web Mobile (DWWM). Il s'agit de concevoir et développer une application web complète simulant le système d'information d'une concession automobile premium fictive.

Le projet couvre l'intégralité du cycle de développement : analyse du besoin, conception, développement frontend et backend, sécurisation des accès et mise en production — compétences directement évaluées lors de l'examen de certification en août 2026.

Objectif général

Fournir une plateforme web qui permet à une concession automobile de gérer son catalogue de véhicules, de traiter les demandes de réservation des clients, et d'administrer l'ensemble via un back-office sécurisé. Le tout accessible depuis n'importe quel navigateur moderne, sans installation côté client.

Périmètre

Le projet couvre uniquement le périmètre web (SPA + API REST). Une application mobile native, un système de paiement en ligne et un module de facturation automatisée sont hors périmètre pour cette version.

Modèle de données principal

Table Champs principaux Description
vehicles id, brand, model, year, price, fuel_type, transmission, mileage, power, description, status, images, created_at Catalogue des véhicules. Statut : available | reserved | sold
reservations id, vehicle_id, client_id, status, message, rdv_date, created_at Demandes de réservation. Statut : pending | confirmed | cancelled
profiles id, role, full_name, created_at Profils utilisateurs liés aux comptes Supabase Auth. Rôle : client | admin

Architecture de déploiement

1
Client (navigateur) : La SPA React compilée est servie statiquement par Express depuis client/dist/. Toutes les routes non-API retournent index.html (mode SPA).
2
Serveur Express (Node.js) : Expose les routes API sous le préfixe /api. Vérifie les tokens JWT sur chaque requête protégée. Tourne sur le port 3001 (ou variable d'environnement PORT).
3
Supabase (cloud) : Héberge la base PostgreSQL, le service d'authentification (JWT) et le stockage des images. Le serveur y accède via la clé de service ; le client via la clé anonyme.
4
Gmail SMTP (Nodemailer) : Relais d'envoi pour le formulaire de contact. Credentials stockés en variables d'environnement, jamais dans le code source.
