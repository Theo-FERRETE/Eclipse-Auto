01 — Contexte
Présentation du projet

Contexte et origine

Eclipse Auto est un projet réalisé dans le cadre de la formation Développeur Web et Web Mobile (DWWM). Il s'agit de concevoir et développer une application web complète simulant le système d'information d'une concession automobile premium fictive.

Le projet couvre l'intégralité du cycle de développement : analyse du besoin, conception, développement frontend et backend, sécurisation des accès et mise en production — compétences directement évaluées lors de l'examen de certification en août 2026.

Objectif général

Fournir une plateforme web permettant à une concession automobile de gérer son catalogue de véhicules, de traiter les demandes de réservation des clients, et d'administrer l'ensemble via un back-office sécurisé. Le tout accessible depuis n'importe quel navigateur moderne, sans installation côté client.

Périmètre

Le projet couvre uniquement le périmètre web (SPA + API REST). Application mobile native, paiement en ligne et facturation automatisée sont hors périmètre.

06 — Merise
Modélisation des données

La modélisation suit la méthode Merise en trois niveaux : conceptuel (MCD), logique (MLD) et physique (MPD). L'application gère trois entités principales : PROFIL, VEHICULE et RESERVATION.

Représente les entités métier et leurs associations, indépendamment de toute technologie.

AssociationCardinalité PROFILCardinalité RESERVATION / VEHICULESignification
effectue (0,n) (1,1) Un profil peut effectuer 0 à plusieurs réservations. Une réservation appartient à exactement 1 profil.
concerne (1,1) (0,n) Une réservation concerne exactement 1 véhicule. Un véhicule peut être concerné par 0 à plusieurs réservations.

MPD — Modèle Physique de Données

Implémentation SQL réelle sur PostgreSQL via Supabase. Les UUID sont générés automatiquement, l'authentification est déléguée à auth.users de Supabase.

-- Table des profils utilisateurs
CREATE TABLE profiles (
  id               UUID        PRIMARY KEY REFERENCES auth.users(id),
  first_name       TEXT,
  last_name        TEXT,
  phone            TEXT,
  role             TEXT        NOT NULL DEFAULT 'client',
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Table des véhicules
CREATE TABLE vehicles (
  id               UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  brand            TEXT        NOT NULL,
  model            TEXT        NOT NULL,
  year             INTEGER     NOT NULL,
  price            NUMERIC     NOT NULL,
  fuel_type        TEXT        NOT NULL,
  transmission     TEXT        NOT NULL,
  mileage          INTEGER     DEFAULT 0,
  power            TEXT,
  status           TEXT        NOT NULL DEFAULT 'available',
  images           TEXT[],
  description      TEXT,
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Table des réservations
CREATE TABLE reservations (
  id               UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  vehicle_id       UUID        NOT NULL REFERENCES vehicles(id),
  client_id        UUID        NOT NULL REFERENCES profiles(id),
  status           TEXT        NOT NULL DEFAULT 'pending',
  rdv_date         DATE,
  message          TEXT,
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now()
);
